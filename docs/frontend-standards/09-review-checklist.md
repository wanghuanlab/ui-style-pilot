# 09 审查清单

本清单用于页面上线前自查、AI 生成后复查、代码评审和历史页面迁移评估。

## 视觉 Token

- [ ] 是否使用 `01-design-tokens.md` 和 `src/root.scss` 中的主色、功能色、中性色。
- [ ] 是否存在未说明原因的硬编码颜色。
- [ ] 页面标题是否为 18px。
- [ ] 区块标题是否为 16px。
- [ ] 正文是否为 14px。
- [ ] 表格、筛选、分页是否使用 12px/14px 的统一层级。
- [ ] 间距是否符合 4/8/16/24/48。
- [ ] 控件圆角是否以 4px 为主。

## 布局

- [ ] 是否符合“面包屑、页面标题、筛选区、工具栏、表格、分页”的标准顺序。
- [ ] 页面背景是否为浅灰，内容容器是否为白底。
- [ ] 筛选区、工具栏、表格边缘是否对齐。
- [ ] 1920x1080 下是否稳定。
- [ ] 1280x720 下关键操作是否仍可见。
- [ ] 页面是否避免全局横向滚动。
- [ ] 表格列多时是否使用表格内部滚动，而不是裁切关键操作。

## 导航

- [ ] 顶部导航高度是否接近 60px。
- [ ] 侧边导航宽度是否接近 240px，收起是否接近 64px。
- [ ] 当前菜单项是否有明确选中态。
- [ ] 当前页面的父级菜单是否展开。
- [ ] 面包屑是否准确反映页面层级。

## 公共组件

- [ ] 是否优先复用了现有公共组件。
- [ ] 是否重复实现了搜索区。
- [ ] 是否重复实现了表格工具栏。
- [ ] 是否重复实现了分页。
- [ ] 是否重复实现了状态标签。
- [ ] 是否重复实现了列配置。
- [ ] 标准动作按钮是否优先使用 `UspActionButton`。
- [ ] 表单和属性区是否优先使用 `UspFormSection`、`UspFormGrid`、`UspFormItem`、`UspReadonlyField`、`UspFormActions`。
- [ ] 新增通用能力是否沉淀为公共组件。
- [ ] 已文档化 `Usp*` 组件的 API、交互或用途变更是否同步更新组件规范。
- [ ] 表格列配置是否优先使用 `UspColumnConfigDrawer`，而不是页面内重复实现。

## 交互

- [ ] 查询按钮是否为主按钮。
- [ ] 新建、编辑、删除、导入、导出、列配置、保存、取消等同类按钮是否在不同页面保持一致。
- [ ] 属性名和值之间的冒号、label 宽度、横向间距、纵向间距是否由组件统一控制。
- [ ] 重置是否恢复默认条件。
- [ ] 更多字段是否不会遮挡关键操作。
- [ ] 依赖选中行的按钮未选中时是否禁用或提示。
- [ ] 危险操作是否二次确认。
- [ ] 提交、同步、导出是否有 loading 或反馈。
- [ ] 空状态是否给出下一步动作。
- [ ] 错误态是否说明原因和处理方式。

## 编码

- [ ] 是否没有新增内联 `style`。
- [ ] 是否没有在页面中手写 `字段名：`、`div + span` 属性展示或 `a-row/a-col + a-form-item` 表单范式。
- [ ] 是否没有新增随意硬编码颜色。
- [ ] 是否没有新增非规范字号。
- [ ] 是否没有在业务页面大面积覆盖 `.ant-*`。
- [ ] 新页面是否避免混用 Ant Design Vue 和 Element Plus。
- [ ] scoped 样式是否只处理页面局部布局。
- [ ] class 命名是否语义化。

## AI 生成自查

- [ ] AI 是否先读取规范文档。
- [ ] AI 是否先搜索公共组件。
- [ ] AI 是否先阅读相关组件规范文档。
- [ ] AI 是否说明复用了哪些组件。
- [ ] AI 是否说明新增了哪些样式和原因。
- [ ] AI 是否按本清单完成自查。

# 02 布局规则

后台页面以清晰、高密度、可扫描为目标。布局应优先服务业务操作，不使用营销式大 Hero、装饰卡片堆叠或过大的留白。

## 页面结构

标准后台结构：

```text
顶部导航
左侧菜单 | 右侧内容区
```

右侧内容区标准顺序：

```text
面包屑
页面标题
筛选区
工具栏
表格/主体内容
分页
```

复杂页面可扩展为：

```text
面包屑
页面标题 + 页面级操作
主内容区
  左侧树/筛选
  右侧列表/详情
```

## 设计基准与适配

- 设计基准：`1920x1080`
- 必须可用：`1280x720`
- 内容区在 1280 宽下不得出现关键按钮不可见、筛选按钮不可达、表格关键列完全被裁切的问题。
- 表格列较多时，可以启用表格内部横向滚动，但页面整体不应横向滚动。

## 页面间距

推荐间距：

| 位置 | 间距 |
|---|---:|
| 内容区外边距 | 12px/16px |
| 面包屑到标题 | 8px |
| 标题到筛选区 | 12px/16px |
| 筛选区到工具栏 | 12px/16px |
| 工具栏到表格 | 12px |
| 表格到分页 | 8px/12px |

紧凑列表页优先使用 12px，表单或详情页可使用 16px/24px。

## 内容容器

- 页面背景使用浅灰底。
- 卡片、表格、弹窗、抽屉使用白底。
- 筛选区可以使用浅蓝底或白底加边框。
- 不要把页面大区块做成多层嵌套卡片。
- 卡片边缘、表格边缘、筛选区边缘应对齐。

## 栅格

- 常规后台使用 12 栅格。
- 信息密集或复杂筛选使用 24 栅格。
- 常用水槽：8px、16px、24px。
- 表单字段宽度应按栅格或固定宽度统一，不要每个字段写不同宽度。

## 搜索区布局

简单筛选：

```text
[字段] [字段] [字段] [查询] [重置]
```

复杂筛选：

```text
第一行：常用字段 + 查询 + 重置 + 更多字段
第二行：展开字段，或使用 Popover/Drawer 承载
```

1280 宽下，查询和重置按钮必须可见。字段过多时优先折叠到“更多字段”，不要让右侧按钮被裁切。

## 表格布局

- 表格应有明确数据对象。可使用页面标题代替，但复杂页面建议表格左上有区块标题。
- 主表行高约 36-42px。
- 紧凑表行高约 28-36px。
- 表头使用浅蓝底和主色文字时，应保持全项目一致。
- 分页位于表格下方，右对齐或随组件统一。

UI Style Pilot 是一套前端 UI 规范治理工具包，将设计标准、Usp 组件模式与审查流程转化为可执行材料，解决 AI 快速生成页面带来的样式漂移与组件重复造轮子问题。

Core Features

核心价值，开箱即用

不只是文档，而是让人类和 AI 都能遵循的完整治理体系。

📐

完整前端规范体系

Token、布局、导航、交互、编码、图表等 12+ 篇文档，统一团队视觉与实现边界。

🧩

Usp 组件模式库

UspPage、UspSearchPanel、UspDataTable 等 Pattern 文档，列表/表单/详情页不再各写一套。

🤖

AI Skills 工作流

页面构建、设计审查、组件复用、样式迁移四大 Skill，让 AI 生成前先读规范、生成后自查。

📋

页面模板 + 审查清单

按列表/表单/抽屉等模板搭骨架，提交前用 Checklist 做一致性验收，减少 Review 摩擦。

🎨

设计源映射

将品牌色、字号、间距映射为后台 Token，降低设计稿到代码的「翻译损耗」。

⚡

轻量预览应用

Vite + Vue 3 落地页，零重型依赖，Fork 即用，GitHub Pages 自动部署。

Tech Highlights

值得开发者关注的设计决策

规范即上下文，治理嵌入流程，而非事后补课。

01

规范 → Skill → 审查 闭环

文档、AI 提示词模板、Skill 定义三位一体，形成可重复的执行路径。

02

组件成熟度分级

draft / stable / deprecated / legacy 四级标记，支持渐进式迁移与试用。

03

B 端场景深度覆盖

列表+筛选+表格+列配置、表单分区+页签、左树右表、状态语义色等高频模式一应俱全。

04

Markdown 即 Agent 上下文

开源、可版本管理，直接作为 Cursor / Copilot 等工具的读取材料，无需额外平台。

05

CI + Pages 工程化

推送即构建校验、自动部署预览，协作与演示门槛极低。

Quick Start

三步上手

Fork 仓库，复制规范到你的项目，让 AI 先读文档再写代码。

terminal 
# 1. 克隆并启动预览
git clone https://github.com/wanghuanlab/ui-style-pilot.git
cd ui-style-pilot
npm install
npm run dev

# 2. 复制规范到你的 Vue 项目
cp -r docs/frontend-standards/ your-project/docs/
cp -r docs/ai-skills/ your-project/docs/

# 3. 让 AI 生成页面前先读这些文件
docs/frontend-standards/USAGE.md
docs/frontend-standards/components/README.md
docs/frontend-standards/09-review-checklist.md
Use Cases

典型适用场景

为 AI 时代的前端团队而生，也适合任何需要 UI 一致性的 B 端项目。

👥

前端团队建立规范

Tech Lead 将零散的设计约定沉淀为可执行文档，减少 Code Review 中反复纠样式与组件结构。

🚀

AI 辅助开发治理

开发者挂载 Skill 后，AI 生成页面自动对齐 Usp 组件与 Token，产出可直接合入主干。

🏢

企业后台数字化

没有完整 Design System 团队，也能获得企业级后台的一致性与可维护性基线。
